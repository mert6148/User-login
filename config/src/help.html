Başlangıç Kılavuzu

İlk Adımlar

  1. Network Manager'a giriş yapın
  2. Ana dashboard'dan sistem durumunu kontrol edin
  3. Settings sayfasından ML konfigürasyonlarını yapın
  4. Cihazlarınızı ekleyin ve yapılandırın
  5. Ağ bağlantılarınızı oluşturun

Hızlı Konfigürasyon

AI/OS konfigürasyonları için hızlı erişim:

  • ML Ayarları: Settings → Model Yönetimi
  • OS Ayarları: Devices → OS Konfigürasyonları
  • Ağ Ayarları: Networks → Ağ Konfigürasyonları
  • Proje Ayarları: Projects → Proje Konfigürasyonları

AI/ML Konfigürasyonu

ML Model Yönetimi

Makine öğrenmesi modellerini yönetmek için:

  1. Settings sayfasına gidin
  2. "Model Yönetimi" sekmesini seçin
  3. Yeni model yüklemek için drag & drop kullanın
  4. Mevcut modelleri test edin veya güncelleyin

Analiz Parametreleri

ML analiz parametrelerini ayarlamak için:

  • Anomali Eşik Değeri: Anomali tespiti için minimum skor
  • Güven Seviyesi: Tahminler için minimum güven yüzdesi
  • Tahmin Penceresi: Tahmin yapılacak zaman aralığı
  • Derin Analiz: Daha detaylı analiz modu

OS Yapılandırması

İşletim Sistemi Ayarları

Farklı işletim sistemleri için konfigürasyon:

Linux Sistemler

  • Ubuntu, CentOS, Debian desteği
  • Paket yönetimi ve güncellemeler
  • Servis yönetimi ve otomasyon

Windows Sistemler

  • Windows Server 2019/2022 desteği
  • PowerShell script yönetimi
  • Grup politikaları ve güvenlik

macOS Sistemler

  • macOS Monterey ve Ventura desteği
  • Homebrew paket yönetimi
  • Sistem tercihleri yönetimi

Ağ Yönetimi

Ağ Konfigürasyonu

Ağ yönetimi ve konfigürasyon rehberi:

LAN Konfigürasyonu

  • Yerel ağ IP adresi yönetimi
  • Subnet mask ve gateway ayarları
  • DHCP ve DNS konfigürasyonu

WAN Konfigürasyonu

  • Geniş alan ağı bağlantıları
  • Bant genişliği yönetimi
  • Yönlendirme ve NAT ayarları

VPN Konfigürasyonu

  • SSL/TLS VPN kurulumu
  • Uzak erişim yapılandırması
  • Güvenlik protokolleri

Sorun Giderme

Yaygın Sorunlar ve Çözümler

ML Servisi Başlamıyor

Çözüm: Settings → Genel Ayarlar → ML Servis Durumu'nu kontrol edin ve gerekirse yeniden başlatın.

Cihaz Bağlantı Hatası

Çözüm: Cihazın IP adresini ve ağ bağlantısını kontrol edin. Firewall kurallarını gözden geçirin.

Model Yükleme Hatası

Çözüm: Model dosyasının formatını (.h5, .pkl, .onnx, .pb) ve boyutunu (max 500MB) kontrol edin.

Ağ Performans Sorunları

Çözüm: Bant genişliği kullanımını kontrol edin ve gerektiğinde QoS ayarlarını yapılandırın.

API Dokümantasyonu

REST API Endpoints

Network Manager API'sini kullanarak sistem yönetimi yapabilirsiniz:

GET /api/v1/health

Sistem sağlık durumunu kontrol eder

POST /api/v1/auth/login

Kullanıcı girişi yapar

GET /api/v1/devices

Tüm cihazları listeler

GET /api/v1/networks

Ağ yapılandırmalarını listeler

Örnek Kullanım

// Health check
fetch('http://localhost:5000/api/v1/health')
  .then(response => response.json())
  .then(data => console.log(data));

// Device list
fetch('http://localhost:5000/api/v1/devices', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
})
  .then(response => response.json())
  .then(data => console.log(data));
